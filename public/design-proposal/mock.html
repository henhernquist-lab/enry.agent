Proposed Token & Spacing Changes

Token / Pattern Current Value Proposed Change Why
--radius 0.375rem (6px) → 0.625rem (10px) base, 0.875rem (14px) for cards Reference uses generous rounding. Current 6px feels hard/utilitarian. 10-14px reads more modern without going bubbly.
Card padding p-4 / p-5 (16-20px) → p-6 (24px) standard for content cards Reference cards have generous internal space. Current 16px feels cramped for content-heavy cards like the briefing.
Card border-radius rounded-lg (8px) → rounded-xl / rounded-2xl (14-16px) Consistent with the wider radius proposal. Makes cards feel like distinct surfaces, not flat panels.
Sidebar width 280px → 260px Slightly narrower because icon+label+desc is more vertically efficient than the current full-width button stack. Net more room for content.
Sidebar nav items Full-width bordered buttons, icon+label only, centered → Rounded nav rows with icon + label + one-line description, left-aligned, grouped by section headers Reference pattern. Descriptions add context without tooltips. Section headers (Navigate / Tools / Research / System) create information hierarchy.
Top bar SystemStatusStrip (monospace data line, 10px, full width) → Cleaner top bar: logo + breadcrumb (page title + description) left, utility icons + status pill right Current status strip is dense data (models ready, resource count, sync time) crammed into 10px mono. The same info moves into a compact status pill; page context gets the real estate.
Card hover border-primary/30 + bg-surface-elevated → border-primary/15 + subtle translateY(-1px) lift Softer border shift (15% vs 30%) + a 1px lift gives a tactile "this is interactive" signal without the current neon-ish green border flash.
Suggestion cards 36px glyph box, 16px padding, border-only → 40px glyph box (rounded-10px), 20px padding, rounded-14px, hover lift More touchable, more breathing room. The rounded glyph box is a small but noticeable polish detail from the reference.
Right panel cards Flat sections with ▸ prefix label, no card container → Each section wrapped in a rounded-12px card with border, consistent with main content cards Unifies the card language across the whole layout. Currently the right panel feels like a different design system than the main area.
Section headers (sidebar) None — flat list with border-t dividers → Labeled section headers: "Navigate", "Tools", "Research", "System" in muted mono uppercase Reference groups nav items with subtle headers. Creates scannable structure instead of a flat 7-item button stack.
Font: section headers font-mono 12px uppercase tracking-wider → font-mono 10px uppercase tracking-[0.1em] at 50% opacity Section headers should recede, not compete. Smaller + more transparent = hierarchy without noise.
--primary (green) #3a9e60 KEPT — no change Enry's brand color. The reference's pink/red is NOT adopted.
--surface-base / secondary / elevated #080808 / #111111 / #171717 KEPT — no change The three-level surface system is solid and already matches the reference's dark-mode philosophy.
--border #262626 KEPT — no change Current border contrast is appropriate. Widening the radius is what changes the feel, not the border color.
